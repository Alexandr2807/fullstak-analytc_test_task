# Task 2 — Публикация товара на маркетплейсе

## User Story

Как продавец,  
я хочу опубликовать свой товар на маркетплейсе,  
чтобы покупатели могли видеть его на витрине и оформлять заказ.

---

## Use Case: Публикация товара

### Наименование
Публикация товара на маркетплейсе

### Цель
Дать продавцу возможность создать и опубликовать карточку товара на витрине маркетплейса.

### Основной актор
Продавец

### Предусловия
- Продавец авторизован в личном кабинете.
- Продавец имеет доступ к разделу управления товарами.
- У продавца есть права на создание товара.

### Постусловия
- Карточка товара сохранена в системе.
- Товар опубликован на витрине маркетплейса и доступен покупателям.

---

## Основной сценарий

1. Продавец переходит в раздел управления товарами.
2. Продавец нажимает кнопку "Добавить товар".
3. Система открывает форму создания товара.
4. Продавец заполняет обязательные поля:
   - наименование товара;
   - описание товара;
   - категорию;
   - цену;
   - фотографии товара.
5. Продавец может заполнить дополнительные поля:
   - артикул;
   - бренд;
   - характеристики;
   - остаток на складе.
6. Продавец нажимает кнопку "Опубликовать".
7. Система выполняет валидацию введенных данных.
8. Система сохраняет карточку товара.
9. Система присваивает товару уникальный идентификатор.
10. Система публикует товар на витрине маркетплейса.
11. Система показывает продавцу сообщение об успешной публикации.

---

## Альтернативные сценарии

### A1. Не заполнены обязательные поля
1. Продавец нажимает кнопку "Опубликовать".
2. Система обнаруживает, что не заполнены обязательные поля.
3. Система не публикует товар.
4. Система отображает сообщения об ошибках по соответствующим полям.

### A2. Загружены некорректные изображения
1. Продавец загружает файл неподдерживаемого формата или слишком большого размера.
2. Система отклоняет загрузку изображения.
3. Система показывает сообщение об ошибке.
4. Продавец загружает корректное изображение.

### A3. Цена указана некорректно
1. Продавец вводит нулевую или отрицательную цену.
2. Система показывает ошибку валидации.
3. Система не дает опубликовать товар, пока цена не исправлена.

### A4. Ошибка на стороне сервера
1. Продавец нажимает кнопку "Опубликовать".
2. Система не может сохранить товар из-за внутренней ошибки.
3. Система показывает сообщение: "Не удалось опубликовать товар. Попробуйте позже."

---

## Бизнес-правила

- Наименование товара обязательно для заполнения.
- Описание товара обязательно для заполнения.
- Категория обязательна для заполнения.
- Цена должна быть больше 0.
- У товара должна быть хотя бы одна фотография.
- Изображения должны быть допустимого формата.
- Только авторизованный продавец может публиковать товар.
- После успешной публикации товар становится доступен на витрине.

---

## Нефункциональные требования

- Время публикации товара после отправки формы — не более 3 секунд при штатной работе системы.
- Сообщения об ошибках должны быть понятны пользователю.
- Система должна предотвращать публикацию невалидных данных.
- Система должна логировать ошибки публикации товара.

---

## Опционально: краткое описание процесса

Продавец заполняет карточку товара в личном кабинете →  
система валидирует данные →  
при успешной валидации сохраняет товар →  
публикует товар на витрине →  
продавец получает подтверждение об успешной публикации.  

---

## Возможные улучшения функциональности

- Черновик товара (сохранение без публикации)
- Модерация товара перед публикацией
- Массовая загрузка товаров
- Предпросмотр карточки перед публикацией
